#  防篡改部署文档

## 上传镜像并上传仓库

###  标记上传

```shell
#加载后端和前端镜像
docker load <wpserver.tar 
docker load < wpweb.tar
#打tag并上传远程仓库
##前端
docker tag wpweb:1.0 192.168.1.88:5000/wpweb:1.0 && docker push 192.168.1.88:5000/wpweb:1.0
##后端
docker tag wpserver:1.0 192.168.1.88:5000/wpserver:1.0 && docker push   192.168.1.88:5000/wpserver:1.0 #注：192.168.1.88为本地仓库需根据部署环境ip仓库
```

### 查看是否成功

出现大概如下：


## 部署 es 、postgresql、wpweb、 wpserver 

###  部署

 web-protect目录下分别有如下4个文件依次部署：
es.yaml   （elasticsearch部署）

 postgres-dep.yaml  （数据库文件）

web-dep.yaml  （后端部署文件）

 webui.yaml  （前端部署文件）

```shell
kubectl apply -f  es.yaml 
kubectl apply -f  postgres-dep.yaml
kubectl apply -f  web-dep.yaml 
kubectl apply -f  webui.yaml  
```

### 查看部署状态

```shell
kubectl get pod -n es
kubectl get pod -n es -A |grep web-protect
```

### 服务端口

```shel
kubectl get svc -A
```
